# 盲人端社区功能实现

## 功能概述

盲人端社区功能允许视障人士加入社区，获得更多帮助和支持。功能包括：

1. **加入社区**：搜索社区ID并加入
2. **社区信息显示**：显示已加入社区的信息
3. **社区功能**：求助、活动、成员等功能（待开发）

## 文件结构

### 布局文件
- `fragment_blind_community_no_join.xml` - 未加入社区时的界面
- `fragment_blind_community_joined.xml` - 已加入社区时的界面

### 数据模型
- `BlindCommunityJoinRequest.java` - 盲人加入社区请求数据模型

### 功能实现
- `CommunityFragment.java` - 盲人社区功能主Fragment

## 功能特点

### 无障碍设计
- 简化UI布局，适合视障人士使用
- 大按钮和清晰的文字标签
- 蓝色主题色，符合应用整体风格

### 用户体验
- 自动检测用户是否已加入社区
- 根据状态显示不同界面
- 友好的错误提示和成功反馈

### API集成
- 使用 `/api/user/blind/community/join` 接口加入社区
- 使用 `/api/community/community/get/id/vo` 接口获取社区信息
- 集成用户信息管理工具

## 使用流程

1. **未加入社区**：
   - 显示欢迎卡片和搜索功能
   - 用户输入社区ID进行搜索
   - 显示社区信息并确认加入

2. **已加入社区**：
   - 显示社区信息卡片
   - 提供求助、活动、成员等功能入口
   - 功能按钮待后续开发

## 技术实现

### 视图切换逻辑
- 基于 `communityId` 字段判断用户状态
- 使用 `UserInfoManager` 获取用户信息
- 动态加载对应的布局文件

### 错误处理
- 网络请求失败提示
- 输入格式验证
- 登录状态检查

### 数据管理
- 使用 `SharedPrefsUtil` 管理登录状态
- 使用 `UserInfoManager` 管理用户信息
- 使用 `RetrofitClient` 处理网络请求

## 待开发功能

1. **求助功能**：发起求助请求
2. **社区活动**：查看和参与社区活动
3. **社区成员**：查看社区成员列表
4. **退出社区**：离开当前社区

## 注意事项

- 确保用户已登录才能使用社区功能
- 社区ID必须是纯数字格式
- 加入社区后需要刷新用户信息
- 所有功能按钮都有相应的提示信息
