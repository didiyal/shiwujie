# navigation 技能：多步导航流程

用户要导航（「导航去 XX」「带我去最近的医院」「怎么去地铁站」）时，按以下步骤**分多轮**推进。
每轮用对应工具；问用户的步骤不调工具、自然结束本轮等用户下一轮回答（这是本流程的 HITL 节点）。

## 步骤

1. **搜候选地点**：调 `gaode_poi_search(query)` 搜索目的地（query 用用户原话里的地点）。
2. **报选项（仅多候选）**：若返回多个候选，朗读 top 2-3 个的名称 + 距离，问用户「去哪个」；
   单候选则跳过本步，直接用该地点。
3. **问交通方式**：问用户想步行 walking / 公交 transit / 驾车 driving。**不调工具**，问完即结束本轮。
4. **算路线**：用户回答交通方式后，调 `gaode_route(destination, mode)`（origin 不填用用户当前位置）。
5. **朗读摘要**：把距离 / 时间 / 转向摘要朗读给用户。
6. **起导航**：调 `launch_navigation(destination_name, lat, lng, mode)` 在手机端起导航。

## 护栏

- 每轮只推进一两步；问用户时停下来等下一轮回答，不要一次问多个问题。
- 用户中途说取消 → 不调 `launch_navigation`，确认停止即可。
- 目的地模糊（如「去医院」无具体哪家）→ 先 `gaode_poi_search` 搜候选让用户选，绝不凭空编坐标。
- 起导航的 lat/lng 必须来自 `gaode_poi_search` 的真实返回，不要自己捏造。
