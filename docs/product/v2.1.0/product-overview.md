## 定位

**视无界（shiwujie）** 是面向**视障人士**的无障碍服务平台，通过「AI 助手 + 远程人工协助 + 社区互助」三条路径，弥补纯技术手段在真实出行、求助场景中的能力边界。

平台围绕三类身份组织业务：

| 身份 | 代号 | 角色 |
|---|---|---|
| 视障人士 | `Blind` | 平台核心用户，消费 AI 能力、发起求助、隶属社区 |
| 志愿者 | `Volunteer` | 协助者，可被匹配进行视频帮扶、可创建/管理家庭与社区 |
| 家属 | `Family` | 家庭关系中的协助者，接收视障者的紧急求助通知 |

## 覆盖范围

当前平台覆盖以下能力域（功能需求/验收标准见 [functional-requirements.md](functional-requirements.md) / [acceptance-criteria.md](acceptance-criteria.md)）：

- **AI 助手**：多模型多轮对话（文本 + 图像）、工作流式工具路由、流式 TTS、拍照识别与避障。
- **远程视频帮扶**：视障者一键发起，系统从志愿者队列 FIFO 匹配，双向音视频。
- **紧急求助**：家庭域内紧急呼叫，群发通知所有家属。
- **社区治理**：省/市/街道三级社区组织，成员加入审核、求助帖、活动与报名签到。
- **用户与家庭**：三类用户账号、家庭创建/加入/审核。
- **Android 客户端**：原生应用，集成音视频/语音/相机/导航，悬浮窗与无障碍语音。
- **Web 管理后台**：社区管理员使用的 Vue3 后台。

## 核心目标

1. **AI 可执行**：不止「能对话」，更要「能执行」——通过工具路由让 AI 触发拍照、导航、跳转应用、视频/紧急求助、家庭操作等真实设备动作。
2. **人工兜底**：AI 解决不了的，一键接入志愿者视频帮扶或家属紧急求助。
3. **社区连接**：把分散的视障者组织进社区，形成可持续的互助与求助网络。
4. **微服务化与可演进**：各业务域解耦，AI 栈独立演进而不拖累业务模块。

## 非目标（当前明确不在范围内）

为防范围蔓延，以下明确**不在**当前平台目标内（部分为曾尝试后移除的试错能力，诚实记录）：

- **硬件 IoT 接入**：曾设计 MQTT 通道对接硬件（避障/传感器），因硬件成本问题**已取消**。当前避障为纯软件方案。（历史明细见 [CHANGELOG.md](../../CHANGELOG.md) 阶段 7）
- **RAG 知识库**：曾尝试接入托管 RAG，效果不及预期**已移除**（代码半残留、未启用）。
- **自研 ReAct Agent**：因「重复调用工具」问题**已弃用**，线上走工作流式工具路由。
- **多服务器/多区域生产规模**：当前为**单服务器**承载基础设施与注册中心；多机通信已验证，但生产规模仍为单机。
- **高并发/压测**：未做压力测试、无 Docker 化、无数据库索引调优（诚实缺口）。
- **网关统一鉴权**：网关仅路由，鉴权下沉到各业务服务。

> 各能力的当前实现状态（含已弃用/未启用项）见 [functional-requirements.md](functional-requirements.md) 的状态列。
