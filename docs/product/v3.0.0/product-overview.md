# 产品概览（v3.0.0 · 进行中）

> 双线推进版本：① 工程架构单体化（**已落地**）；② AI 助手重写（**设计敲定·实现待 Phase 5**）。产品定位、三类身份、覆盖范围**继承 v2.1.0**（见 [../v2.1.0/product-overview.md](../v2.1.0/product-overview.md)，已封版 2026-07-11）。本文件属 `product/v3.0.0/`（当前工作版本）。版本分级模型见 [CONTRIBUTING.md](../../CONTRIBUTING.md) 第五节；当前版本指针 [../current.md](../current.md)。

## 本版本焦点：双线推进

v3.0.0 同时推进两条线：

### 线 ① · 单体化改造（已落地）

反思 v2.1.0 的 Spring Cloud 微服务对当前体量**过度设计**，v3.0.0 去微服务、按当前规模精简为单体应用（大方向见 [../../ROADMAP.md](../../ROADMAP.md) v3.0.0 段）。

> **单体化工程已于 2026-07-11 落地（启动级验证通过）**：合并为单体应用、统一技术栈、合库、跨服务调用改本地（交付明细见 [../../development/v3.0.0/task-breakdown.md](../../development/v3.0.0/task-breakdown.md)；架构现状见 [../../architecture/tech-stack.md](../../architecture/tech-stack.md)）。

- **架构（已落地）**：合并 user / call / community / ai 为单体应用（后端收敛为契约层 + 唯一应用模块两模块、统一技术栈），消除版本割裂与鉴权逻辑多处重复。
- **用户可见契约（零变更）**：API 路径 / WebSocket 信令 / FR / AC 继承 v2.1.0——启动级回归通过，功能级待 App/Web 联调。实际变更（如入口/端口收敛）记录于本目录 [functional-requirements.md](functional-requirements.md) / [acceptance-criteria.md](acceptance-criteria.md) / [changelog.md](changelog.md)。
- **能力补全**：App 集成高德 SDK（替代 v2.1.0 的 URI 调起）。
- **安全加固**：v2.1.0 收尾遗留的 🔴 安全项（ai 默认用户后门、删改权限检查、WS 鉴权、密码哈希、弱密钥、前端 TLS）在单体化语境下统一修复。

### 线 ② · AI 助手重写（设计敲定·实现待 Phase 5，未落地）

平台的 AI 助手将由当前的工作流式问答，**整体重写为「智能体驱动」**——助手能连续进行多轮对话，并在对话中**主动调用设备与平台能力**为视障用户代办事务，包括：拍照识别、联网查询（含带定位的天气）、导航（搜目的地、报候选、询问交通方式、起导航）、加入/退出/查询家庭、修改个人信息、发起紧急求助与志愿者视频通话、跳转白名单应用。助手还会**记住用户偏好**（说话方式、常用应用、导航习惯等），并在多轮对话中自动保持上下文不丢失、必要时压缩历史。

> **状态：设计已敲定（Phase 1–4 梳理完成），实现待 Phase 5，尚未落地。** 本条目及本目录相关 FR/AC/changelog 条目均为**设计契约**，不代表功能已上线。设计细节见 [../../development/v3.0.0/](../../development/v3.0.0/)；选型理由见 [../../architecture/tech-stack.md](../../architecture/tech-stack.md)。

用户可见面要点（设计敲定）：

- **对话通道**：AI 对话从单向文字流回复，改为与求助信令**同一双向通道**（WebSocket `/api/ws/call`）——盲人用户面对的是一条统一的、可双向收发的对话与求助通道，而非割裂的多个入口。
- **求助信令码不变**：求助信令 `5001–5006` 的语义与编码沿用 v2.1.0；其中导航信令 `5006` 首次具备携带目的地信息（名称 / 经纬度 / 地址）的能力（加可选字段，不破坏旧字段名）。
- **业务功能契约零变更**：用户、家庭、社区、求助帖、视频帮扶等业务功能的接口、字段、状态码**全部不变**；本次重写仅触及 AI 对话通道与求助信令载荷，属本次重写范围内的同步改造，非违约。
- **安全门（设计即定）**：紧急求助须用户确认后才发起；个人信息修改仅限基本字段（昵称、手机号、性别等），助手对密码、身份证、残疾证等敏感字段拒绝并引导专门入口；跳转应用仅限白名单（通讯、生活类）。

> 实施前置确认（如所用模型的工具调用稳定性）属开发交付范畴，记录于 [../../development/v3.0.0/](../../development/v3.0.0/)；若前置确认不通过，相关 FR/AC 以本目录为准回退调整。

## 非目标（沿用 v2.1.0）

见 [../v2.1.0/product-overview.md](../v2.1.0/product-overview.md#非目标当前明确不在范围内)，v3.0.0 不改变这些边界。因单体化而**不再适用**的工程项（分布式事务 Seata、网关统一鉴权、Knife4j 聚合 ai）见 [../../ROADMAP.md](../../ROADMAP.md) 删除线标注。AI 助手重写**不新增用户可见业务实体**（用户/家庭/社区/求助帖等业务对象与契约不变），仅重写 AI 助手的对话与代办能力。
