# 功能需求（v3.0.0 · 进行中）

> 单体化改造版本。用户可见功能需求**继承 v2.1.0**（完整 `FR-<MODULE>-<NN>` 清单见 [../v2.1.0/functional-requirements.md](../v2.1.0/functional-requirements.md)，已封版 2026-07-11）。本版本聚焦单体化重构，**原则上不改用户可见业务功能**；因单体化可能产生的契约变更（如入口/端口收敛）待架构定稿后在此补 FR 编号。

## 继承基线

AI 助手 / 视频帮扶 / 紧急求助 / 社区治理 / 用户家庭的全部 `FR-*` 继承自 v2.1.0，不在此重复（去重铁律：FR/AC 只在 product 出现一次）。

## v3.0.0 新增 / 变更

### 社区域权限边界（安全加固，2026-07-12）

- **FR-COMMUNITY-PERM-01**：平台须对求助帖删除/修改、社区修改/删除、社区管理员增删改执行授权校验——求助帖删改限作者或本社区注册人/管理员；社区修改/删除仅限注册人；社区管理员管理仅限注册人/管理员（员工无权），并保护唯一注册人不被降级/删除、不新增第二个注册人。越权请求返回业务码 `NO_AUTH`（40030）。

### 用户口令存储（安全加固，2026-07-12）

- **FR-AUTH-PWD-01**：用户口令须以 BCrypt 加盐哈希存储，不以无盐 MD5 存储；存量无盐哈希在首次正确登录时自动升级。

### AI 助手重写功能基线（设计敲定·实现待 Phase 5，未落地）

> 平台的 AI 助手将由工作流式问答重写为「智能体驱动」的对话助手。以下 `FR-AI-NEW-*` 为**设计契约**，实现待 Phase 5，尚未落地。仅盲人用户可用 AI 助手；志愿者/员工侧不涉及。业务实体（用户/家庭/社区/求助帖/视频帮扶）的契约零变更，本段仅描述 AI 助手的能力与用户可见行为。功能清单继承 v2.1.0 的 AI 助手能力面并重写实现，不在此重复旧编号。

#### 拍照识别

- **FR-AI-NEW-01 拍照即识别**：盲人用户进入应用默认处于摄像头取景状态，一键拍照后助手对图片进行识别，返回**语音友好的描述**（简短、可朗读、突出对盲人有用的事物）；该图片进入当前对话上下文，用户可继续追问。

#### 语音多轮对话

- **FR-AI-NEW-02 多轮连续对话**：助手支持连续多轮对话，跨轮保持上下文（如对刚拍的图片追问、记住本轮提到的地名），不因换轮而丢失上文；上下文过长时助手自动压缩较早历史，近期对话不被压缩。
- **FR-AI-NEW-03 图片追问**：用户可在拍照识别后继续以语音追问图片内容。
- **FR-AI-NEW-04 软件功能介绍**：助手能介绍本应用自身的功能与用法（基于内置功能说明文档）。
- **FR-AI-NEW-05 常识问答（联网 + 带定位天气）**：助手能回答常识性问题，可联网检索最新信息；天气类问题须结合用户当前位置作答。
- **FR-AI-NEW-06 软件代办（我的位置）**：用户可询问「我在哪里」，助手朗读当前位置地址。
- **FR-AI-NEW-07 软件代办（拍照）**：用户可在对话中要求助手拍照（与拍照按钮等价的语音入口）。
- **FR-AI-NEW-08 软件代办（跳转应用）**：用户可在对话中要求助手跳转到白名单应用（通讯类、生活类）；助手对非白名单应用拒绝跳转。
- **FR-AI-NEW-09 软件代办（加入家庭）**：用户可通过对话加入家庭。
- **FR-AI-NEW-10 软件代办（紧急求助）**：用户可通过对话发起紧急求助，但**须经用户确认**后才实际发起紧急求助信令（见 FR-AI-NEW-16 安全门）。
- **FR-AI-NEW-11 软件代办（志愿者通话）**：用户可通过对话发起志愿者视频通话。
- **FR-AI-NEW-12 软件代办（修改个人信息）**：用户可通过对话修改个人基本字段（昵称、手机号、性别）；助手对密码、身份证号、残疾证号等敏感字段拒绝修改并引导专门入口（见 FR-AI-NEW-17 安全门）。
- **FR-AI-NEW-13 社区代办（暂缓）**：社区相关代办能力（如发帖、评论）本期**暂缓**，不纳入 AI 助手能力面。

#### 导航（分步确认）

- **FR-AI-NEW-14 导航分步流程**：用户说出目的地后，助手执行分步流程：① 搜索目的地并**报出候选地点**供用户选择；② 用户确认目的地后，助手**询问交通方式**（步行 / 公交 / 驾车）；③ 助手按所选方式规划路线、朗读路线摘要；④ 起导航。其中「确认目的地」「询问交通方式」为**人工确认点**，助手不得跳过确认直接起导航。

#### 记忆

- **FR-AI-NEW-15 偏好记忆**：助手在用户不可见的后台**自动**抽取并维护偏好（说话方式、常用应用、导航习惯等软事实），跨会话生效；偏好注入后续对话作为参考，但用户的当次言行**优先于**历史偏好（可覆盖）；偏好维护不得阻塞或打断对话。

#### 安全门（设计即定）

- **FR-AI-NEW-16 紧急求助确认门**：紧急求助须用户**显式确认**后才发起紧急求助信令（`5003`）；误触、对话中被误判、或助手自行判断，均**不得**直接发起紧急求助。
- **FR-AI-NEW-17 个人信息修改字段门**：助手可修改的个人字段**仅限**昵称、手机号、性别等基本字段；密码、身份证号、残疾证号等敏感字段**结构性不可达**（非提示词约束），助手对涉及敏感字段的请求拒绝并引导专门入口。
- **FR-AI-NEW-18 跳转应用白名单**：助手可跳转的应用仅限白名单（通讯类、生活类）；非白名单应用跳转请求被拒绝。

#### 操作入口

- **FR-AI-NEW-19 应用入口与四按钮**：盲人用户进入应用即处于摄像头取景状态，提供 4 个操作按钮：①**拍照识别**（一键拍照并进入识别）；②**语音输入**（进入对话）；③**我的位置**（仅朗读当前位置地址，不进入导航）；④**操作说明**（静态朗读固定的帮助说明）。四个按钮各自行为独立、互不混淆。

### 通信通道变更（设计敲定·实现待 Phase 5，未落地）

> 以下为 AI 助手重写对用户可见通信通道的影响，属本次重写范围内的同步改造，**非违约**（AI 对话通道本就整体重写）。App 须同步改造以对接。状态：设计敲定·实现待 Phase 5，尚未落地。

- **FR-AI-NEW-20 对话通道改双向**：AI 对话从单向文字流回复（旧对话通道），改为与求助信令**同一双向通道**（WebSocket `/api/ws/call`）——文本对话与求助信令共用一条可双向收发的通道；旧的 AI 对话通道（`/api/ai/ai/{doChatByText,doChatByImage,NewApp}`）随重写废弃清除。
- **FR-AI-NEW-21 求助信令码不变**：求助信令 `5001–5006` 的语义与编码沿用 v2.1.0，**零变更**；其中导航信令 `5006` 首次具备携带目的地信息（目的地名称 / 经纬度 / 地址，可选字段、不破坏旧字段名）的能力。WebSocket 全部信令码（`-1/0/1/2/3/4/5001~5006`）框架不变。
- **FR-AI-NEW-22 业务功能契约零变更**：用户、家庭、社区、求助帖、视频帮扶等业务功能的 HTTP 接口、字段、HTTP/业务状态码**全部不变**；本次重写不触及这些业务契约。
