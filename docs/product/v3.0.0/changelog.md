### AI 助手重写（设计敲定·实现待 Phase 5，未落地）

> 以下为 AI 助手重写的**用户可见变更**，属本次重写范围内的同步改造，非违约（AI 对话通道本就整体重写）。**全部条目状态：设计已敲定（Phase 1–4），实现待 Phase 5，尚未落地。** 对应 `FR-AI-NEW-*` / `AC-AI-NEW-*` 见 [functional-requirements.md](functional-requirements.md) / [acceptance-criteria.md](acceptance-criteria.md)。

- **变更（设计）· 对话通道改双向**：AI 对话从单向文字流回复，改为与求助信令同一双向通道（WebSocket `/api/ws/call`）；旧 AI 对话通道 `/api/ai/ai/{doChatByText,doChatByImage,NewApp}` 随重写废弃清除。盲人用户面对一条统一的、可双向收发的对话与求助通道。
- **变更（设计）· 导航信令携带目的地**：求助信令 `5001–5006` 编码与语义不变；其中导航信令 `5006` 首次具备携带目的地信息（名称 / 经纬度 / 地址，可选字段、不破坏旧字段名）的能力。
- **变更（设计）· 应用入口四按钮**：盲人用户进入应用即处于摄像头取景状态，提供 4 个操作按钮——拍照识别 / 语音输入 / 我的位置（仅朗读当前位置）/ 操作说明（静态朗读帮助）。
- **变更（设计）· 导航分步确认**：导航改为分步流程（报候选 → 确认目的地 → 询问交通方式 → 朗读摘要 → 起导航），含「确认目的地」「询问交通方式」两个人工确认点。
- **变更（设计）· 偏好记忆**：助手在后台自动维护跨会话的偏好（说话方式、常用应用、导航习惯等），用户当次言行可覆盖历史偏好。
- **不变（设计）· 求助信令码与业务契约**：求助信令 `5001–5006` 编码零变更；用户/家庭/社区/求助帖/视频帮扶等业务功能的接口、字段、状态码**全部不变**。
- **安全门（设计）**：紧急求助须用户确认后才发起；个人信息修改仅限基本字段（昵称/手机号/性别），敏感字段（密码/身份证/残疾证）拒绝并引导专门入口；跳转应用仅限白名单。
