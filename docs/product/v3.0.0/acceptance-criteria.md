# 验收标准（v3.0.0 · 进行中）

> 单体化改造版本。用户可见验收标准**继承 v2.1.0**（完整 `AC-<MODULE>-<NN>` 清单见 [../v2.1.0/acceptance-criteria.md](../v2.1.0/acceptance-criteria.md)，已封版 2026-07-11）。本版本聚焦单体化重构，**原则上不改用户可见业务功能**；因单体化产生的契约变更待架构定稿后在此补 AC 编号。

## 继承基线

AI 助手 / 视频帮扶 / 紧急求助 / 社区治理 / 用户家庭的全部 `AC-*` 继承自 v2.1.0，不在此重复（去重铁律：FR/AC 只在 product 出现一次）。

## v3.0.0 新增 / 变更

### 社区域权限边界收紧（安全加固，2026-07-12）

> 此前任意登录用户可删/改任意求助帖与社区、随意增删社区管理员；本版本收紧授权边界（HTTP 路径 / 业务码不变，仅授权判定收紧）。越权请求返回业务码 `NO_AUTH`（40030）。

- **AC-COMMUNITY-PERM-01 求助帖删改**：删除/修改求助帖仅限①该帖作者（视障者本人）或②其所属社区的注册人/管理员；社区员工与其他用户无权，返 `NO_AUTH`。
- **AC-COMMUNITY-PERM-02 社区修改/删除**：修改/删除社区仅限该社区的**注册人**；其他用户（含管理员/员工）无权，返 `NO_AUTH`。
- **AC-COMMUNITY-PERM-03 社区管理员管理**：新增/修改/删除社区管理员仅限该社区的注册人或管理员（员工无权）；且①不可新增第二个注册人、②不可降级或删除该社区唯一的注册人（末位注册人护栏），违者返 `NO_AUTH`。
- **AC-COMMUNITY-PERM-04 删除管理员按目标**：删除社区管理员按请求指定的目标成员删除（此前实现误删为调用者自己，已修正）。

### 用户口令安全存储（安全加固，2026-07-12）

- **AC-AUTH-PWD-01 口令哈希存储**：用户口令以 BCrypt 自适应哈希（带盐）存储，不再以无盐 MD5 存储；存量用户首次以原口令登录通过后自动升级，对用户无感（登录/注册/改密端点与请求/响应字段不变）。

### AI 助手重写验收（设计敲定·实现待 Phase 5，未落地）

> 对应 [functional-requirements.md](functional-requirements.md)「AI 助手重写功能基线」「通信通道变更」。以下 `AC-AI-NEW-*` 为**设计验收标准**，实现待 Phase 5，尚未落地。仅盲人用户侧。

- **AC-AI-NEW-01 拍照识别语音友好描述**：盲人用户拍照后，助手返回的识别结果是**语音友好的描述**（简短、可朗读、突出对盲人有用的事物），而非冗长原始标注或不可朗读的格式。
- **AC-AI-NEW-02 多轮对话不丢上下文**：跨轮对话中，助手能引用本轮之前出现过的信息（如刚拍的图片内容、本轮提到的地名）；上下文过长时较早历史被压缩，但近期对话内容不丢失。
- **AC-AI-NEW-03 导航分步含人工确认点**：用户说出目的地后，助手依次完成「报出候选地点 → 用户**确认目的地** → **询问交通方式**（步行 / 公交 / 驾车）→ 朗读路线摘要 → 起导航」；其中「确认目的地」与「询问交通方式」为人工确认点，助手在用户未确认前**不得**直接起导航。
- **AC-AI-NEW-04 偏好跨会话生效且可覆盖**：助手记住的偏好跨会话生效（如常用导航方式）；但用户当次的言行优先于历史偏好——例如用户历史偏好驾车但本次说「我步行去」，助手按步行处理。
- **AC-AI-NEW-05 我的当前位置按钮**：点击「我的位置」按钮，助手**仅朗读**当前位置地址，**不进入**导航流程。
- **AC-AI-NEW-06 操作说明按钮**：点击「操作说明」按钮，助手**静态朗读**固定的帮助说明（非动态生成）。
- **AC-AI-NEW-07 语音输入按钮进对话**：点击「语音输入」按钮进入对话状态（可多轮交互）。
- **AC-AI-NEW-08 拍照按钮直连识别**：点击「拍照识别」按钮直接拍照并进入识别，不经过对话中间步骤。
- **AC-AI-NEW-09 紧急求助确认门**：发起紧急求助信令（`5003`）**必须经用户显式确认**；误触、对话中被误判、或助手自行判断，均不得直接发起紧急求助信令。
- **AC-AI-NEW-10 个人信息修改字段门**：助手可修改的字段**仅限**昵称、手机号、性别等基本字段；对涉及密码、身份证号、残疾证号等敏感字段的修改请求，助手**拒绝**并**引导**用户到专门入口（如改密页）。
- **AC-AI-NEW-11 跳转应用白名单**：助手仅可跳转白名单应用（通讯类、生活类）；非白名单应用的跳转请求被拒绝。
- **AC-AI-NEW-12 对话通道双向**：AI 对话在求助信令同一双向通道（WebSocket `/api/ws/call`）上双向收发；旧的单向文字流回复通道（`/api/ai/ai/{doChatByText,doChatByImage,NewApp}`）不再存在。
- **AC-AI-NEW-13 求助信令码与业务契约不变**：求助信令 `5001–5006` 编码与语义沿用 v2.1.0；导航信令 `5006` 在携带可选目的地信息（名称 / 经纬度 / 地址）时不破坏旧字段名；用户/家庭/社区/求助帖/视频帮扶等业务功能的接口、字段、状态码全部不变。
