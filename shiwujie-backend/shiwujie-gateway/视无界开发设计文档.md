# 视无界重构设计

# 一，产品需求分析

产品内容站在使用者角度进行分析

## 视障人士

1. 先下载软件，然后注册账号

   1. 支持手机号一键注册，账号密码注册
   2. 一键注册成功后需设置密码才可以账户密码登录

2. 注册成功后选择身份我是视障人士

3. 进入软件

4. 首先要输入残疾人证件信息，选择加入社区与家庭

   1. 残疾人证件信息正常输入
   2. 加入社区
      1. 加入当地官方社区
   3. 加入家庭

5. 功能使用：

   1. AI功能
      1. 环境识别
      2. APP操作
      3. 外部跳转
      4. 悬浮窗
   
   2. 求助志愿者
      1. 主动区分**本地求助**与**全网求助**（是否公开位置信息）
      2. **点击向空余志愿者求助**
      3. 优先匹配距离近的志愿者
      4. 若30s无志愿者自动转接到紧急求助
      5. **求助完成后有评价部分**
   
   3. 紧急求助
      1. 向家属发送求助信息，视频求助(后续可以加入电话求助)
      2. 可以设置除家庭外的求助人
   4. 社区
      1. 文字预约求助
      2. 社区活动展示（展示社区与附近的活动）展示市
      3. 最近的新闻信息，政策福利
      4. 残疾人交流
   5. 个人信息修改（家庭）
   
   

> [!IMPORTANT]
>
> 问题：
>
> 1. 加入社区如何选择？
>    1. 居委会、村委会
> 2. 是否可以不加入社区？那志愿活动怎么开展？
>    1. 可以，不加入不能使用线下功能
>    2. 若这样视障人士归类到哪里？当地的未加入社区视障人群中？
>       1. 自动归并到市残联区里面



## 志愿者（家属）

1. 先下载软件，然后注册账号

   1. 支持手机号一键注册，账号密码注册
   2. 一键注册成功后需设置密码才可以账户密码登录
   3. 后续可以支持微信与其它登录
2. 注册成功后选择身份我是志愿者
3. 进入软件



#### 志愿者

1. 加入社区
2. 加入或者创建家庭（可选）
3. 功能使用：
   1. 加入求助电话，可以视频帮助视障人士
      1. 等待中可以操作其它事情
   2. 社区**（社区志愿者可用）**
      1. 接收**社区附近**的求助（一个区内）
      2. 加入活动（本市所有活动）
         1. 活动签到功能
         2. 活动评价
   3. 个人信息修改（家庭）

> 5. 志愿者是否在注册后需要选择自己的位置信息，用来区分地区？默认定位
